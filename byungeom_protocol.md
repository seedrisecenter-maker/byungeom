# 變臉(변검) 正反合 프로토콜 v4.0
# Claude Code 직접 실행 — MAX 요금제 전용, 추가 과금 $0

## 사용법
사용자가 다음과 같이 요청:
```
"正反合: [요청 내용]"
또는
"변검: [요청 내용]"
```

## 프로세스

### 正(정) — 설계자
1. 요청 분석 → 복잡도 판정 (LOW/MID/HIGH)
2. 계약서(Contract) 작성: 함수명, 입출력, 제약조건, 엣지케이스
3. 검증용 assert 테스트 작성
4. 구현 지시서 작성

### 反(반) — 구현자
1. 계약서 기반 코드 구현
2. 자기 약점 1가지 명시
3. Bash로 즉시 실행 검증

### 合(합) — 검증자
1. 계약 충족 여부 확인
2. 正의 테스트케이스 실행
3. 엣지케이스 공격
4. 판정: PASS / PATCH / FAIL
5. PATCH → 反 재구현 (최대 3회)
6. FAIL → 正부터 재설계

### 완료
- PASS된 최종 코드를 파일로 저장
- 요약 보고

## 장점
- 추가 과금: $0 (MAX 요금제 내)
- 모델: Opus 4 (orchestrator의 Haiku보다 월등)
- 토큰 낭비: 0 (시스템 프롬프트 반복 없음)
- 코드 실행: Bash 직접 실행 가능
- 파일 접근: Read/Write/Edit 가능
